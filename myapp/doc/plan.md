# CtoCサイト「grand-market」開発計画書

## 1. 概要

本ドキュメントは、CtoCサイト「grand-market」の開発計画を定めるものです。
既存の認証設計 (`auth-plan.md`) とビルド手順 (`build.md`) を踏まえ、サイトに必要なコア機能と開発の進め方を定義します。

## 2. プロジェクトの現状

- **技術スタック:** Spring Boot/Kotlin, TypeScript/Tailwind CSS
- **認証:** OAuth, 二段階認証(TOTP)を実装予定。設計は `auth-plan.md` に記載済み。
- **ビルド:** `build.md` に手順が定義済み。

## 3. 主要機能一覧

開発する主要な機能を以下に示します。
各機能の詳細は、リンク先の個別ドキュメントを参照してください。

1.  **ユーザー機能:**
    - 詳細は [user-features.md](./user-features.md) を参照。
    - ユーザープロフィール、評価システムなど。
    - 認証機能は `auth-plan.md` に基づき実装。

2.  **商品出品・一覧機能:**
    - 詳細は [listing-features.md](./listing-features.md) を参照。
    - 商品の出品、検索、カテゴリ分類、詳細表示など。

3.  **取引機能:**
    - 詳細は [transaction-features.md](./transaction-features.md) を参照。
    - 商品購入、決済、ユーザー間メッセージングなど。

4.  **管理・運営機能:**
    - 詳細は [admin-features.md](./admin-features.md) を参照。
    - 不正利用の監視、コンテンツモデレーション、問い合わせ対応など。

## 4. 開発フェーズ（案）

1.  **フェーズ1: 基本基盤の実装**
    - ユーザー登録・ログイン機能（`auth-plan.md` に基づく）
    - 商品一覧表示・詳細表示
    - 商品出品フォーム

2.  **フェーズ2: 取引機能の実装**
    - 商品購入フロー
    - 決済システム連携（ダミーから開始）
    - ユーザー間メッセージ機能

3.  **フェーズ3: 付加価値機能の実装**
    - ユーザー評価機能
    - 商品検索機能の高度化
    - 管理機能

## 5. 次のステップ

本計画書に基づき、各機能の詳細設計と実装を進めます。
まずはフェーズ1の「ユーザー登録・ログイン機能」と「商品一覧表示」から着手することを推奨します。
